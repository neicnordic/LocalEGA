Service Description Usage Status
db A Postgres database with appropriate schema, stores the file header the accession id, file path and checksums as well as other relevant information. Submission and Data Retrieval
mq A RabbitMQ message broker with appropriate accounts, exchanges, queues and bindings. We use a federated queue to get messages from CentralEGA's broker and shovels to send answers back. Submission
Inbox SFTP server, acting as a dropbox, where user credentials come from Central EGA Submission
Ingest Split the Crypt4GH header and move it to database and move the remainder to the storage backend (archive). No cryptographic task required. Submission
Verify Using a crypt4gh secret key can decrypt the stored files and checksum them against their embedded checksum. Submission
Archive Storage backend: as a regular file system or as a S3 object store. Submission and Data Retrieval
Finalize Also called handles the so-called Accession ID (stable ID) to filename mappings from CentralEGA store. Submission
Data Out API Provides a download/data access API for streaming archived data either in encrypted or decrypted format. Data Retrieval
Metadata Component used in standalone version of SDA. Provides an interface and backend to submit Metadata and associated with a file in the Archive. Submission Data Retrieval
